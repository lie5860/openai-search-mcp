# Frontend Development Guidelines

> Best practices for frontend development in this project.

---

## Overview

**This project has no frontend.** It is an MCP (Model Context Protocol) server that runs on Node.js and communicates via stdio. The guidelines in this directory are **placeholders** for the day you add a web UI or other frontend; until then, AI agents should rely on `spec/backend/` and `spec/guides/` only.

## When Adding a Frontend Later

1. Document your project's **actual conventions** (not ideals).
2. Include **code examples** from your codebase.
3. List **forbidden patterns** and why.
4. Add **common mistakes** your team has made.

The goal is to help AI assistants and new team members understand how YOUR project works.

---

**Language**: All documentation should be written in **English**.
